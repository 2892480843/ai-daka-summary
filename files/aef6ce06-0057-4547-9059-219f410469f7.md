# 交互式地图工具 · 规划文档（PLAN）

> 项目代号：`cc260620_map_label`
> 首个用例：**中国春秋·南通动物园** 导览图（`map_pic/yc_dt_big.jpg`）

---

## 1. 项目定位

一个**通用的交互式地图构建工具**：管理端标注/配置 → 用户端浏览。
动物园地图只是第一个项目，未来可扩展到园区、展览、商场导览、校园地图等。
**数据模型与界面均不绑定具体场景**——换场景只换数据和底图，代码不变。

---

## 2. 总体架构

- **本地单机**，纯前端（HTML/CSS/JS），**零后端、零依赖、零构建**。
- **两个界面**：
  - `admin.html` — 管理端（标注 + 配置，**可读写**）
  - `viewer.html` — 用户端（浏览，**只读**）
- **共享数据层**：两端通过同一个本地服务器运行，共享浏览器 **IndexedDB**。
  管理端存进去的，用户端立刻能读出来。
- 需本地静态服务器（**不能用 `file://` 双击**，IndexedDB/fetch 不可靠）：
  `python -m http.server 8000`，访问 `localhost:8000/admin.html`、`/viewer.html`。

```
        浏览器（同一个 localhost，共享 IndexedDB）
┌─────────────────────────────┬─────────────────────────────┐
│    管理端 admin.html         │     用户端 viewer.html       │
│  标注 + 配置（可读写）        │   浏览（只读）                │
└──────────────┬──────────────┴──────────────┬──────────────┘
               │        db.js (IndexedDB)     │
               └───────────────┬──────────────┘
                               ▼
                 Maps → Regions → Types → Images(blob)
```

## 3. 数据模型（通用）

### Map（一个地图项目）
```
{ id, name, bgImageId, width, height, settings, createdAt, updatedAt }
```

### Region（一个可点击点位）
```
{
  id, mapId,
  name,        // "熊猫馆"
  typeId,      // 引用全局类型库（animal / performance / ...）
  shape,       // { type:'rect', x, y, w, h }（预留 polygon 升级）
  content: { text, images[], extra },   // 介绍 / 配图 / 趣闻·场次
  tags[],      // 细标签，如"食肉类""亲子推荐"
  order, createdAt
}
```

### Type（全局类型库，可配置）⭐ 重点
```
{ id, name, color, icon, builtin }
```
- **全局共享**：所有地图复用同一套类型定义，不绑定单张地图。
- **数据驱动**：管理端"类型管理"面板可增 / 删 / 改，设名称、颜色、图标。
- **预置内置类型**：动物 🐼、表演 🎪、游乐设施 🎡、服务设施 ℹ️。
- **无限扩展**：后期加任意类型（如餐饮、卫生间、停车场、表演场…），**无需改代码**。

### Image（底图 / 配图，blob 存储）
```
{ id, blob, createdAt }
```

## 4. 管理端工作流

1. 新建 / 选择地图项目 → 上传底图（存 IndexedDB）
2. **画框**（拖拽矩形）→ 每框自动编号入队
3. 点"开始识别" → **LLM 三步**：识名 → 分类 → 生成介绍
4. **校对编辑**（表单可改名称/类型/介绍/趣闻/标签）
5. 保存到 IndexedDB
6. **类型管理**（随时配置全局类型库）

## 5. 用户端工作流

1. 选择一个地图
2. 看底图 → 不同类型用**不同颜色/图标**的小标记
3. hover 高亮 → 点击 → **弹出介绍卡片**
4. 只读，不能改

## 6. LLM 接入（可配置适配层）

```
管理端设置面板：
  · 后端类型：ollama（默认）/ llama.cpp / OpenAI 兼容（留作将来接云端）
  · endpoint：http://localhost:11434
  · model：minicpm-v / qwen2.5-vl / gemma3 等（管理员填）
  · Prompt 模板：可查看可改（高级）
```

**识图三步流程**：
```
画框 → 裁小图 → base64 → 发给 VLM
   Prompt: "识别这个区域：①名称 ②类型(动物/表演/游乐/服务/其他) ③图上原文"
   → { name, type 候选, rawText }
   → 若 type=动物：再让 LLM 按 name 生成 { 介绍, 趣闻 }
   → 管理员校对 → 保存
```

**已知风险**：
- 卡通图标 + 本地小模型 → 识别 / 分类可能不准，**校对环节必须**（结果做成可编辑表单）。
- **CORS**：浏览器请求 `localhost:11434` 被拦，需 `setx OLLAMA_ORIGINS "*"` 后重启 ollama（工具内置失败提示）。

## 8. 分阶段路线

- **阶段 1（当前）**：数据层 + 管理端（传图→画框→LLM 三步→校对→保存）+ 用户端（加载→按类型显示→点击看介绍）。**数据结构从第一天就按通用多类型设计**。
- **阶段 2**：多地图项目管理、导入/导出 JSON（备份迁移，为将来转"导出部署"留口子）、用户端类型筛选。
- **阶段 3**：多边形区域、移动端适配、缩放平移、图鉴收集、可能的话音朗读。

## 9. 关键决策记录

| 维度 | 决策 |
|------|------|
| 部署 | 本地单机，双界面（admin/viewer），IndexedDB 共享，零后端 |
| 区域形状 | 矩形框（数据结构预留 polygon 升级） |
| 类型 | **全局类型库，数据驱动，可配置扩展**（不写死） |
| LLM | 本地 ollama/llama.cpp；识名 + 分类 + 生成介绍 三步；接入可配置 |
| 内容来源 | LLM 根据识别出的名称生成介绍，管理员校对 |
| 区域粒度 | 以动物为主，含表演 / 游乐 / 服务等多类型点位 |
