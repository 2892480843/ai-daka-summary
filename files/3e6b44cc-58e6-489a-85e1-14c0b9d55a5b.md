# READ Code Search Web Application

A single-page HTML web application for searching through a medical READ codes database containing **81,219 records**.

## 🎯 Features

- **Single Page Application** - Complete standalone application, no build tools or frameworks
- **Embedded Data (v2)** - All 81,219 READ codes embedded directly — no file loading required
- **Full-Text Search** - Search across READCODE, READV3CODE, and TERM columns (case-insensitive)
- **Data Grid Display** - Professional table with all three columns
- **Pagination** - Navigate through large result sets (20 items per page)
- **Modern UI** - Clean, responsive design with gradient colors
- **No Server Required** - Works directly in your browser via `file://` protocol
- **XSS Protection** - All user-facing output is HTML-escaped

## 🚀 Quick Start

### Method 1: Embedded Data Version (Recommended)

1. Double-click **`index-v2.html`** to open it in your browser
2. The data loads automatically — no file selection needed
3. Start searching immediately!

### Method 2: File-Picker Version (Alternative)

Use `index.html` if you prefer to load the CSV file manually:

1. Double-click `index.html` to open it in your browser
2. Click the **"Choose CSV File"** button
3. Navigate to and select `source-data/medtech-read-code-v2.csv`
4. Wait for the success message confirming data is loaded
5. Start searching!

### Method 3: Using Command Line

```bash
# On Windows
start index-v2.html

# On macOS
open index-v2.html

# On Linux
xdg-open index-v2.html
```

## 📋 Usage Instructions

### Searching

1. Type your search term in the search bar (upper center)
2. Click the **"SEARCH"** button or press **Enter**
3. Results will appear in a data grid table below
4. Use **Previous/Next** buttons to navigate through multiple pages of results
5. If nothing is found, a **"No data available"** message is displayed

### Search Examples

- Search by READCODE: `159A.00`
- Search by READV3CODE: `159A.`
- Search by TERM: `diabetes`
- Partial searches work too: `diab` will match "diabetes"
- Search is case-insensitive

## 📊 Data Format

The source data is a pipe-delimited CSV file with the following structure:

```
READCODE|READV3CODE|TERM
159A.00|159A.|Tubal Ligation
1925.00|1925.|Excessive Salivation
...
```

**Dataset stats:** 81,219 records, 3 columns, pipe (`|`) delimited.

## 🛠️ Technical Details

### Two Approaches

| Version | File | Approach | Use Case |
|---------|------|----------|----------|
| **v2 (Recommended)** | `index-v2.html` | CSV data embedded directly as a JavaScript array | Instant access, no file loading |
| **v1** | `index.html` | HTML5 FileReader API to load CSV at runtime | Smaller file size, load different datasets |

### Why Two Versions?

Browsers block JavaScript from fetching local files directly via `fetch()` when opening HTML files from the file system (`file://` protocol) due to CORS (Cross-Origin Resource Sharing) security restrictions.

- **`index-v2.html`** solves this by embedding all data directly in the HTML — no fetch required.
- **`index.html`** solves this by using the HTML5 FileReader API, which allows users to explicitly select a file, then the browser grants permission to read it.

### Regenerating `index-v2.html`

If the source CSV is updated, regenerate the embedded version using:

```bash
# Requires Python 3
python archive/generate_v2_embedded.py
```

This reads `source-data/medtech-read-code-v2.csv` and writes a new `index-v2.html` (~3.9 MB).

## 🔧 Troubleshooting

### `index-v2.html` — Page Loads but No Data

1. Check the browser console (F12) for JavaScript errors
2. The file is large (~3.9 MB) — allow a moment for it to load on slower machines
3. Regenerate the file: `python archive/generate_v2_embedded.py`

### `index.html` — "Error loading data" Message

1. Make sure you're selecting the correct file: `source-data/medtech-read-code-v2.csv`
2. Ensure the CSV file is not corrupted
3. Check that the file has the correct pipe-delimited format

### Search Returns No Results

1. Verify data was loaded successfully
   - `index-v2.html`: Check the green banner showing the record count
   - `index.html`: Check for the "✓ Data Loaded" indicator
2. Try different search terms
3. Check for typos in your search term
4. Remember that search is case-insensitive
