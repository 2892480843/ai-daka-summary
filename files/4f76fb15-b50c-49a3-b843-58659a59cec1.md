# 算法思维反思日志
## ——以王夫之数字人为例

---

### 初始目标与思路（约 150 字）

最开始的目标很直接：构建一个王船山先生的数字人，能和他"对话"，借此理解他的哲学体系，同时把《船山遗书》这部体量庞大的原典"用起来"。

设计思路上，我没有直接说"帮我做一个聊天机器人"，而是先加了一句约束："不要只是寻章摘句，断章取义，写作者必须对船山先生的基本思想、核心观点有清晰的认识，必须不断拷问写作者。"这句话是刻意前置的——它的意图是把任务的质量标准先说清楚，再说任务本身。类比到算法设计：这相当于在写函数之前先定义验收条件，不是先动手再看结果。

---

### 任务拆解与迭代过程（约 250 字）

整个项目大致分为六步，每一步都是在上一步的输出基础上发现新问题、提出新要求。

第一步是确立人格架构：要求模型给出王夫之的核心哲学立场，拿到了"五层架构"（气论、知行论、历史哲学、人性论、处境情感）。第二步是把演示变成工具，一句话"给我一个单独的 HTML 网页文件，在浏览器里打开就能用"，解决了"拿不走"的问题。

第三步是可用性扩展。发现只支持 Anthropic、背景太暗，于是在一条 Prompt 里并列了两个要求："换成可以用其他家 API 的，要包含支持智谱清言 API 的。背景太暗了，换成明代山水画背景。"这里的技巧是把同一层级的修改打包在一条里，避免多轮零碎修改带来的累积错误。

第四步是接入原典。上传 EPUB 后加了一句关键约束："不允许泛化问题，不接受搜索网络信息整合的回答。"这条 Prompt 把数字人从"扮演口吻"升级为"有据可依"。

第五步上传三张真迹图片，加了格式要求"每一句都加上白话文翻译"。第六步是发现模型编造了五条假格言之后，选择"先做核对功能：把检索到的原文显示出来，让我自己判断"——这是一个主动降低自动化程度、把判断权留给人的决策。

---

### 大模型的表现评估（约 150 字）

表现好的地方主要有两处：一是建立哲学骨架。当我明确说"不要寻章摘句，要理解核心观点"之后，模型确实给出了有层次的思想体系而不是语录拼接，这说明"先定标准"的前置约束发挥了作用。二是代码的工程实现，从单文件 HTML 到嵌入知识库、再到流式输出和折叠面板，每轮迭代的代码质量都基本可用，不需要从头重写。

跑偏的地方也很典型：在被问到"该如何对待工作焦虑"时，模型生成了"知足常乐、淡泊明志、道德自律"五条格言，每条都配着工整的白话翻译。问题不是格式错了，而是内容编造了——格式的正确性反而给了假内容一层可信外观，这是"逐句白话"这条格式要求意外带来的副作用，当时没有预料到。

---

### 问题与修复策略（约 200 字）

项目里出现了三类问题，修复策略各不相同。

**第一类：约束太软**。"查不到就说无载"只是一句嘱咐，模型在扮演压力下照样编造。修复思路是加折叠面板：不靠模型自觉，而是把检索到的原文暴露出来，让人来核对。这个策略最有效，因为它改变的不是"模型的意愿"，而是"人能不能发现问题"——绕过了难以控制的一层，转而强化了可以控制的一层。

**第二类：检索精度低**。字面匹配（n-gram）会漏掉语义对应的段落。问"人性会不会变"，原文是"性日生而日成"，意思一致但字面不重叠，结果检索漏掉了。这个问题至今未修，计划接 embedding 接口改成向量检索。

**第三类：三重约束打架**。"只用原典 + 人格鲜活 + 逐句白话"三条规则同时压着，小模型容易顾此失彼——为了凑白话翻译，把船山的血气说没了。修复方式是换大一档的模型，而不是调整 Prompt，这说明有些问题的根源是模型能力，不是提示词设计。

---

### 最终总结（约 200 字）

做这个项目之前，我对"算法思维"的理解偏向技术层面——算法是计算机执行的步骤，和我的日常使用关系不大。做完之后的理解是：算法思维的核心是"把一个模糊的大目标拆解成可以逐步验证的小步骤，每一步都有明确的输入、输出和验收条件"，而这正是和模型协作时必须做的事。

"我想要一个船山数字人"是一个模糊目标，模型无法直接执行。我必须回答：什么叫做合格的数字人（验收条件）、知识从哪来（输入）、用什么方式呈现（输出）、出了问题怎么修（错误处理）。每一个 Prompt 本质上都是在写这个拆解的一个子步骤。

另一个改变是对"迭代"的理解。我以前以为迭代是"改了哪里、变好了"。但这个项目让我意识到：诚实的迭代记录里应该写"意图改善但未实测""已知弱点未解决"——把没做好的地方说清楚，比把每次改动都说成"变好了"更有价值，因为它告诉后来者下一步该做什么。

---

### 作品的真实价值（约 100 字）

会用。下一步最想补的是两件事：一是把字面检索换成语义检索，这样问"人性会变吗"能真正检索到"性日生而日成"；二是补一组覆盖核心立场的测试题——"郡县制好还是封建制好""王阳明的致良知对不对""如何看清朝"——用不同模型跑一遍，看哪里答对、哪里漂移，再有针对性地调整。作品现在有骨架，下一步要做的是验证它。
